◆ COMPARISON_THRESHOLD

final double COMPARISON_THRESHOLD = 1E-14
static

Relative difference tolerated when comparing floating point numbers using ComparisonMode#APPROXIMATE.

Historically, this was the relative tolerance threshold for considering two matrixes as org.geotoolkit.referencing.operation.matrix.XMatrix::equals(Object, ComparisonMode) equal}. This value has been determined empirically in order to allow org.apache.sis.referencing.operation.transform.MathTransforms to detect the cases where two org.apache.sis.referencing.operation.transform.LinearTransform are equal for practical purpose. This threshold can be used as below:

{ java Matrix m1 = ...; Matrix m2 = ...; if (MatrixUtilities.epsilonEqual(m1, m2, EQUIVALENT_THRESHOLD, true)) { // Consider that matrixes are equal. } }

By extension, the same threshold value is used for comparing other floating point values.

3.20
