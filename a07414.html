Detailed Description

An authority factory creating CRS from the { TABLE} table in a spatial SQL database.

This class is called Direct<u>Postgis</u>Factory because of some assumptions more suitable to PostGIS, like the default getAuthority() authority} if none were explicitly defined. But this class should be usable with other OGC compliant spatial database as well.

This factory doesn't cache any result. Any call to a

createFoo

method will trig a new WKT parsing. For adding caching service, this factory needs to be wrapped in a org.geotoolkit.referencing.factory.CachingAuthorityFactory instance. The AuthorityFactoryProvider convenience class can be used for that purpose.

Author
Martin Desruisseaux (Geomatys)

synchronized Integer getPrimaryKey ( final Class<? extends IdentifiedObject >  type,
String  code 
) throws NoSuchAuthorityCodeException, FactoryException

Returns the primary key for the specified authority code.

If the supplied code contains an <cite>authority</cite> part as in

"EPSG:4326"

, then this method searches for a row with the given authority (

"EPSG"

) in the { AUTHORITY_COLUMN} column and the given integer code (

4326

) in the { CODE_COLUMN} column. If such row is found, then the value of its { PRIMARY_KEY} column is returned.

If the supplied code does not contain an <cite>authority</cite> part (e.g.

"4326"

), then this method parses the code as an integer. This is consistent with common practice where the spatial CRS table contains entries from a single authority with primary keys identical to the authority codes. This is also consistent with the codes returned by the getAuthorityCodes(Class) method.

Parameters
typeThe type of the object being created (usually CoordinateReferenceSystem}.class).
codeThe authority code to convert to primary key value.
Returns
The primary key for the supplied code. There is no guarantee that this key exists (this method may or may not query the database).
Exceptions
NoSuchAuthorityCodeExceptionif a code can't be parsed as an integer or can't be found in the database.
FactoryExceptionif an error occurred while querying the database.
