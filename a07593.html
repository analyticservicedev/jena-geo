Detailed Description

A container of factories frequently used together, with utility methods.

This class serves two purpose:

  1. A container for the following factories:
    • DatumFactory
    • CSFactory
    • CRSFactory
    • MathTransformFactory
  2. Utilities methods for creating new CRS derived from an existing one.

{

Note
The CoordinateOperationFactory factory is intentionally excluded from this container, because CoordinateOperationFactory is more a processing class than a factory creating directly objects from the parameters.}
Author
Martin Desruisseaux (IRD, Geomatys)

CoordinateReferenceSystem toGeodetic3D ( final CompoundCRS  crs) throws FactoryException

Returns a CRS equivalents to the given one, with some components replaced by their 3D counterpart when possible.

More specifically, if the given compound CRS contains two consecutive CompoundCRS::getComponents() components CRS} where:

  • One component is a two-dimensional GeographicCRS geographic} or ProjectedCRS projected} CRS;
  • The other component is a one-dimensional VerticalCRS vertical} CRS and its datum type is
    ELLIPSOIDAL
    (height above the ellipsoid)

Then this method replaces those two components by a single three-dimensional component. The other components (for example TemporalCRS temporal} CRS) are included unchanged in the returned CRS.

If there is no (2D + 1D) components that this method could replace by a 3D component, then this method returns the

crs

argument unchanged. In any case, the transform from the given CRS to the returned CRS shall be an identity transform.
