Detailed Description

A set of RGB colors created by a PaletteFactory palette factory} from a name.

A palette can creates an IndexColorModel index color model} or an ImageTypeSpecifier image type specifier} from the RGB colors.

Author

Martin Desruisseaux (IRD, Geomatys)

◆ getLegend()

RenderedImage getLegend ( final Dimension  size) throws IOException
inherited

Returns the color palette as a smoothed image of the specified size, together with the palette name.

The image returned by this method differs from getImage(Dimension) in three ways:

  • The image returned by this method uses a different color model than the images to be produced by this
    Palette
    .
  • The colors in this image are interpolated in order to produce a smooth image. Consequently, some colors in the returned image may not exist in the palette.
  • The palette name is written in the returned image.

WARNING: Some colors in the returned image may be misleading - they may have no geophysical meaning. Consider for instance an image of Sea Surface Temperature with colors ranging from blue (cold) to red (hot). The correct way to interpolate is to first interpolate the temperature, then use the color map for fetching the color associated to that temperature. With this approach, interpolation between cold and hot water will produce warm water, often displayed in yellow or green. But

getLegend

method will produce purple colors - because it performed the interpolation in ARGB space - which is wrong.

◆ lower

final int lower
protected

Index of the first valid element (inclusive) in the IndexColorModel index color model} to be created.

Pixels in the range 0 inclusive to

lower

exclusive will be reserved for "no data" values.

Strictly speaking, this index should be non-negative because IndexColorModel does not support negative index. However this

Palette

implementation accepts negative values provided that upper is not greater than { Short::MAX_VALUE}. If this condition holds, then

Palette

will transpose negative values as positive values in the range

0x80000

to

0xFFFF

inclusive. Be aware that such approach consume the maximal amount of memory, i.e. 256 kilobytes for each color model.
