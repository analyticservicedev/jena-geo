Detailed Description

Watch for file-system modification for a given folder.

There's a recursive mode to allow watch of all sub-directories as well. If the recursive mode is activated, when a new directory is created in the watched one, it will be registered for watching, as all its sub-directories.

Watched folders are registered for the standard event types defined by java.nio.file.StandardWatchEventKinds.

Use of the class :

  • Build a new watcher, with a boolean argument to specify if we want recursive survey (true) or not (false) : final DirectoryWatcher watcher = new DirectoryWatcher(true) {..your implementation...};
  • Give it the separate folders to work on : final Path toWatch1, toWatch2; watcher.register(toWatch1, toWatch2);
  • Your watcher is ready to go, call the start()} method to launch the survey. /!\ It launches an asynchronous survey in a separate thread.
  • To stop|pause the watching, you just have to call stop()} method.
  • Finally, when you're done with survey, please close your watcher using close()}.

N.B : Thread-safe implementation.

It allows user defining its own processes over file changes.

countNumber of times the same event occurred.
