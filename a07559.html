Detailed Description

A group of related parameter values.

The same group can be repeated more than once in an org.opengis.referencing.operation.Operation operation} or higher level ParameterValueGroup, if those instances contain different values of one or more ParameterValues which suitably distinguish among those groups.

ParameterValue<?> parameter ( String  name) throws ParameterNotFoundException

Returns the value in this group for the specified Identifier::getCode identifier code}.

If no ParameterValue parameter value} is found but a ParameterDescriptor parameter descriptor} is found (which may occurs if the parameter is optional, i.e. ParameterDescriptor::getMinimumOccurs minimumOccurs} == 0), then a ParameterValue parameter value} is automatically created and initialized to its ParameterDescriptor::getDefaultValue default value} (if any).

This convenience method provides a way to get and set parameter values by name. For example the following idiom fetches a floating point value for the

"false_easting"

parameter:

{ java double value = parameter("false_easting").doubleValue(); }

This method does not search recursively in subgroups. This is because more than one subgroup may exist for the same ParameterDescriptorGroup descriptor}. The user must groups query all subgroups} and select explicitly the appropriate one to use.
