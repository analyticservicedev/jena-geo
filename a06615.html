Detailed Description

Serializable rectangle capable to handle infinite values.

Instead of using

x

,

y

,

width

and

height

fields, this class uses xmin, xmax, ymin and ymax fields. This choice provides two benefits:

  • Allows this class to works correctly with java.lang.Double::POSITIVE_INFINITY infinites} and java.lang.Double::NaN NaN} values (the width and height alternative has ambiguities).
  • Slightly faster
    contains
    and
    intersects
    methods since there is no addition or subtraction to perform.

Martin Desruisseaux (IRD, Geomatys)

instead, for better support of multi-dimensional data and easier portability to JavaFX or Android platforms.

Tests if the interior and/or the edge of two rectangles intersect.

This method is similar to Rectangle2D#intersects(Rectangle2D) except for the following points:

Tests if the interior of the.

Shape

intersects the interior of a specified rectangle. This method might conservatively return

true

when there is a high probability that the rectangle and the shape intersect, but the calculations to accurately determine this intersection are prohibitively expensive.

This method is similar to Shape#intersects(Rectangle2D), except that it tests also rectangle with zero Rectangle2D::getWidth width} or Rectangle2D::getHeight height} (which are Rectangle2D::isEmpty empty} according Shape contract). However, rectangle with negative width or height are still considered as empty.

This method is said <cite>inclusive</cite> because it try to mimic intersectInclusive(Rectangle2D, Rectangle2D) behavior, at least for rectangle with zero width or height.
