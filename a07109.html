Adds to the given hints only the valid entries from the given map.

This method is similar to

target.putAll(source)

except that invalid keys are filtered. Such filtering is not always wanted since it may hide bugs.

◆ ATTEMPTS_DELAY

final int ATTEMPTS_DELAY = 200
static

The minimal delay (in milliseconds) between unsuccessful calls to Factory#availability.

If a first call failed, then all subsequent calls before the end of this delay will returns

false

immediately without actually attempting a connection to the database. This is used in order to avoid too frequent connection attempts, assuming the reason for the initial failure changed have probably not changed before this delay.
