Detailed Description

A Collections::checkedSet(Set, Class) checked} and Collections::synchronizedSet(Set) synchronized} LinkedHashSet.

The type checks are performed at run-time in addition to the compile-time checks.

Using this class is similar to wrapping a LinkedHashSet using the methods provided in the standard Collections class, except for the following advantages:

  • Avoid the two levels of indirection (for type check and synchronization).
  • Checks for write permission.
  • Overrideable methods for controlling the synchronization lock, type checks and write permission checks.

The synchronization is provided mostly in order to prevent damages to the set in case of concurrent access. It does not prevent java.util.ConcurrentModificationException to be thrown during iterations, unless the whole iteration is synchronized on this set getLock() lock}. For real concurrency, see the java.util.concurrent package instead.

{

Note
The above is the reason why the name of this class emphases the <cite>checked</cite> aspect rather than the <cite>synchronized</cite> aspect of the set.}

◆ checkWritePermission()

void checkWritePermission ( ) throws UnsupportedOperationException
protected

Checks if changes in this set are allowed.

This method is automatically invoked after this set got the getLock() lock} and before any operation that may change the content. If the write operation is allowed, then this method shall returns normally. Otherwise an UnsupportedOperationException is thrown.

The default implementation does nothing significant (see below), thus allowing this set to be modified. Subclasses can override this method if they want to control write permissions.

{

Note
Actually the current implementation contains an assert statement ensuring that the thread holds the lock. This is an implementation details that may change in any future version of the SIS library. Nevertheless methods that override this one are encouraged to invoke super.checkWritePermission().}
