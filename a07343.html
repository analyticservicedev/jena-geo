◆ ignoreNaN()

boolean ignoreNaN ( boolean  ignore)

Indique si les interpolations devront ignorer les NaN dans le vecteur des y.

Par défaut, les NaN ne seront pas ignorés de sorte qu'ils pourront être retournés par la méthode interpolate(double) s'ils apparaissent dans le vecteur des y. Si vous demandez à ignorer les NaN, alors la méthode interpole tentera de toujours utiliser des données valides pour ses calculs. Cette méthode n'a aucun effet sur les vecteur des x. Pour ce dernier, les NaN seront toujours ignorés.

Parameters
ignoretrue si la méthode interpolate(double) doit agir comme si elle utilisait des copies des données dans lesquelles on avait retiré tout les NaN.
Returns
l'ancien état.

Implemented in Table1D.

◆ interpolateAt()

double interpolateAt ( int  index) throws ExtrapolationException

Renvoie la valeur y interpolé à x[index] mais sans utiliser la valeur de y[index].

Cette méthode est utile pour boucher les trous causé par les données manquantes (NaN). Cette méthode est plus rapide que interpole car elle ne nécessite pas que l'on recherche la position de xi dans le vecteur des x.

Vous pouvez aussi utiliser cette méthode pour interpoler des pics isolés qui vous semble suspects. Toutefois s'il y a une possibilité que deux pics soient collés, il est préférable de remplacer tous les pics par des NaN et ensuite d'utiliser cette méthode pour combler les trous.

Parameters
indexindex du x pour lequel on veut interpoler un y.
Exceptions
ExtrapolationExceptionsi une extrapolation non-permise a eu lieu.

Implemented in Table1D, and Search1D.

◆ recompute()

void recompute ( )

Signale à la table que des données du vecteurs des x ou du vecteur des y ont été changées.

Certains types d'interpolations telle que l'interpolation cubique B-Spline ont besoin de savoir s'ils doivent recalculer leurs champs internes. L'implémentation par défaut ne fait rien.

Implemented in Spline1D, and Table1D.
